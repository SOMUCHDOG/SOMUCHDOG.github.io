# How I Scope Side Projects

A side project fails when scope is vague and success criteria are undefined.

## My Process

1. Define one user and one core workflow.
2. Set a strict build window (usually two weekends).
3. Ship the smallest version with observable metrics.

## Guardrails

- No optional features in v1.
- No premature infrastructure complexity.
- Every task must map to user value.

Focused scope is less exciting than big ideas, but it ships.
